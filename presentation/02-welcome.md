# You're doing the work *with* Copilot

<p class="lede">Not just finishing the task — <mark>prompting it, reviewing its output, and refining</mark>. That skill applies whether you write code, user stories, test plans, or security reviews.</p>

## Everyone has a role

| Role | Where you shine |
| --- | --- |
| **Business analyst** | Turn user stories into a crisp spec, write acceptance criteria, keep scope honest |
| **Developer** | Build the feature from the spec, pair with Copilot, wire up CI |
| **Tester** | Drive unit + automated tests, find edge cases, run the accessibility scan |
| **Cybersecurity** | Lead the security review and scanning, threat-model the feature |

## What "done" looks like today

The **core spine (C1–C5)** is what we aim to complete — repo &amp; Copilot setup, stories→spec, build, unit tests, test automation. **Stretch goals (S1–S5)** — security, accessibility, CI/CD, docs, legacy — are bonus and independent. A working C1–C5 beats a half-done stretch goal.

<p class="pixel">This next half hour is the "how". Then you hack.</p>
