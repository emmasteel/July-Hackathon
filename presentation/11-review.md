# Review — build vs. what you asked for

When the build is done, walk the spec top to bottom and flag anything the build *doesn't* do that the spec asked for. Review is where a plausible-looking result gets held to the bar you set.

## Check Copilot's work

- **Understand it before you accept it.** If you can't explain the code, ask Copilot to explain it — then decide.
- **Read carefully** — not just "does it run", but security, readability, maintainability.
- **Let tools help** — unit tests (C4), automated checks (C5), linting, code scanning, the accessibility scan.

<div class="callout tinted">
Real reviews catch real things: a missing button, a desktop layout served to phones, a search result that jumps to the page but not the right section.
</div>
