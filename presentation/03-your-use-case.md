# Same objectives, two use cases

<p class="lede">Both teams build an <strong>accessible, secure, public-facing government web feature</strong>. Only the domain differs.</p>

<ul class="cards">
  <li>
    <span class="tag">Team 1 — ABR</span>
    <h3 style="font-family:inherit">Accessible ABN lookup</h3>
    <p>Analyse the Australian Business Register for accessibility gaps against WCAG 2.2 AA, then build an accessible ABN lookup/results feature that shows how to do it right.</p>
  </li>
  <li>
    <span class="tag">Team 2 — Grant Finder</span>
    <h3 style="font-family:inherit">Accessible grants checker</h3>
    <p>Build an accessible grants / eligibility checker that helps a small business find grants it qualifies for — and explains <em>why</em> each one matches.</p>
  </li>
</ul>

<div class="callout tinted">
Ground rule: never call the live government services. Use the local fixtures only — your app and tests run on sample data.
</div>
