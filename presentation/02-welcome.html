Welcome & framing

You're doing the work with Copilot

Not just finishing the task — prompting it, reviewing its output, and refining. That skill applies whether you write code, user stories, test plans, or security reviews.

Everyone has a role

RoleWhere you shine
Business analystTurn user stories into a crisp spec, write acceptance criteria, keep scope honest
DeveloperBuild the feature from the spec, pair with Copilot, wire up CI
TesterDrive unit + automated tests, find edge cases, run the accessibility scan
CybersecurityLead the security review and scanning, threat-model the feature

What "done" looks like today

The core spine (C1–C5) is what we aim to complete - repo & Copilot setup, stories→spec, build, unit tests, test automation.
Stretch goals (S1–S5) - security, accessibility, CI/CD, docs, legacy - are bonus and independent. A working C1–C5 beats a half-done stretch goal.

This next half hour is the "how". Then you hack.
