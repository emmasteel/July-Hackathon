Core objectives - the shared spine

Do these in order. Each is about completing the task with GitHub Copilot: prompting it, reviewing its output, and refining.

  • C1

    Set up for GitHub Copilot

    Author a copilot-instructions.md and prompt files so GitHub Copilot understands your conventions.

  • C2

    Stories → spec

    Turn user stories into functional and non-functional requirements with acceptance criteria.

  • C3

    Build the feature

    Build the core feature for your use case from the spec.

  • C4

    Unit testing

    Generate, review, and strengthen unit tests with Copilot.

  • C5

    Test automation

    Build an automated end-to-end / regression check.

Stretch goals — bonus, pick any

How the day runs

  1. Arrive, check your setup, run a quick Copilot sanity test.
  2. Welcome, meet your team, get your use case.
  3. Set up the repo for Copilot (C1), then turn stories into a spec (C2).
  4. Build the feature (C3) and strengthen it with unit tests (C4).
  5. Add automated tests (C5), then grab any stretch goals you fancy.
  6. Show-and-tell: demo your feature and your best Copilot moment.

💡 Tips for the day

  • Prompt, review, refine. Copilot is a pair, not an oracle — always read and correct its output.
  • Protect the spine. A finished C1–C5 beats a half-done stretch goal.
  • Everyone contributes. BAs shape the spec, testers drive tests, security leads the review, developers build.
  • Small steps. Break work into focused prompts and ask follow-up questions in Copilot Chat.
  • Stay safe. No secrets, no real data, and never hit live government sites in tests — use the fixtures.
  • Ask for help. Facilitators are there to pair on code or unblock you.

Happy hacking! 🚀
